# Farm

**会話が、真実であり続ける文書になる。**

Farmは、AIとの会話から知的生産物（事業計画、営業資料、LP、仕様書、小説）を育てるための環境設計フレームワークです。

あなたは文書を書きません。AIと話します。会話の要点が正史（soil）に蒸留されて積もり、文書は正史からコンパイルされます。正史が変われば、どの文書が古くなったかを機構が教えてくれます。

名前には2つの意味があります。**農場（Farm）**——人間の意志やペイン（種）を、AIという成長エンジンで作物のように育てる場所。**強固な仕組み（Firm）**——AIの幻覚や暴走を、正しい情報と機構で抑える堅牢なハーネス。

## 思想

AIは工場の機械ではなく、自然物に近い存在です。命令ではなく、環境（文脈・素材・制約・フィードバック）が出力の質を決めます。人間の仕事は執筆ではなく、良い成果物が生まれる環境の設計です。

- AIの出力は完成品ではなく収穫物である
- ドラフトは失敗ではなく作物である
- ボツ案は廃棄物ではなく堆肥である
- プロジェクトは一回で完成しない。出荷・反応・改良を季節として回す

## 設計原則

v1を実プロジェクトで数か月運用して得た教訓を、5つの原則に固定しました。

1. **機構は指示に勝る。** AIに「覚えておいてもらう」散文ルールは腐る。保証したいことはスクリプトとgitで機械的に検査する。
2. **文書はコンパイルする。** 成果物は手で維持しない。正史（soil）から再生成する。同じ主張を複数の文書へ手作業で同期する仕事を、設計から消す。
3. **正史は小さく。** soilは「生成前に全部読める」サイズ（10ファイル目安）に保つ。参照漏れ事故の唯一の根治は、全部読めることである。
4. **儀式は一つだけ。** セッション終わりの蒸留。台帳・変更ログ・索引の手動維持は持たない。履歴はgitが持つ。
5. **メタファーはブランド、構造は退屈に。** 農園の語彙は思想を伝えるためにあり、ディレクトリは4つで足りる。

## 構造

```
0_soil/      土壌 = 正史。プロジェクトの確定情報（10ファイル目安）
1_seeds/     種   = 受信箱。会話の蒸留・素材・外部フィードバック（追記のみ）
2_harvest/   収穫 = 成果物。soilからコンパイルされた文書
9_compost/   堆肥 = 旧版・ボツ案（生成時に読まない）
barn/        納屋 = 道具。スクリプト・ルール・テンプレート
agent.md     AIへの最初のシステム指示（エントリーポイント）
```

数字の接頭辞は**信頼順位**です。AIは0を最も信頼し、9を生成に使いません。

- `0_soil/` — 生成に使ってよいのは `status: canonical` のファイルのみ。各ファイルはfrontmatter（後述）を持つ
- `1_seeds/` — `conversations/`（会話の蒸留）、`materials/`（投げ込み素材）、`feedback/`（外部反応）。分類に迷ったら直下に置いてよい
- `2_harvest/` — 成果物ごとに1ディレクトリ。版番号ディレクトリは作らない（履歴はgitが持つ）
- `9_compost/` — 検索汚染防止のため、生成時には参照しない。再発掘するときだけ見る

## 動作原理 — 4つの可動部品

### 1. 小さな正史（soil）

事業計画もセールスデッキも、正体は「意思決定と主張の集合 + 読者別のレンダリング」です。会話を通じて成長するのは主張の集合（soil）であり、文書はそこからコンパイルされる出力です。

soilの各ファイルは先頭にfrontmatterを持ちます。

```yaml
---
title: 製品定義
summary: 何を・誰に・どう提供するかの正規定義
status: canonical   # canonical | draft | deprecated
updated: 2026-07-03
tags: [product, pricing]
---
```

`barn/farm.sh index` がこのfrontmatterから `0_soil/_index.md` を自動生成します。索引は手で維持しません。ファイル末尾の「関連」リンクをAIが辿ることで、周辺知識を芋づる式に収集します（検索の仕組みは `barn/rules.md` §7）。

### 2. 蒸留の儀式（唯一の必須ルーチン）

意味のある会話をしたセッションの終わりに、AIが必ず行います。

1. 会話の要点を `1_seeds/conversations/YYYY-MM-DD_<topic>.md` に保存する（決定・保留・未解決の問い）
2. 正史への差分を提案し、承認を得て `0_soil/` を直接編集する
3. コミットする。**コミットメッセージが変更ログそのもの**になる

これが「話しているうちに成果物ができていく」の実体です。会話→蒸留→soil差分→影響を受けた文書の再コンパイル、というループだけが回ります。

### 3. コンパイルされる成果物（harvest）

成果物は先頭に依存マニフェストを持ちます。

```yaml
---
artifact: business_plan
audience: 投資家
status: draft        # draft | shipped
depends_on:
  - 0_soil/identity.md
  - 0_soil/claims.md
built_from: a1b2c3d   # ビルド時のsoilのcommit
built_at: 2026-07-03
---
```

soilが変わると、`barn/farm.sh status` がこのマニフェストとgit履歴を突き合わせて「この文書は古い（stale）」と教えます。古くなった文書は差分修正ではなく、soilから再生成します。

### 4. 機構（barn）

```bash
bash barn/farm.sh status   # 農園の健康診断：soilの状態 / staleな成果物 / 未処理seeds / 未コミット変更
bash barn/farm.sh index    # soil索引をfrontmatterから再生成
bash barn/farm.sh stale    # staleな成果物だけを表示
```

セッション開始時にAIが `status` を実行します。AIがルールを思い出す必要はなく、機構が現状を告げます。

## 運用方法

**毎セッション：**

1. 開始 — AIが `farm.sh status` を実行し、staleな成果物と未処理のseedsを報告する
2. 作業 — 話す。soilを直す。成果物をコンパイルする。順番は自由
3. 終了 — 蒸留の儀式。コミット

**出荷前（成果物を外に出すとき）：**

1. 監査の帽子（生成とは別のコンテキスト）で、soilとの整合・主張の根拠・表現をチェックする
2. `git tag ship/<artifact>/<YYYY-MM-DD>` を打つ。出荷版の再現はタグが保証する

**月次（またはstatusが警告したとき）：**

- soilの `updated` が古いファイルを見直し、内容が正しければ日付を更新、古ければ内容を直すか `deprecated` にする

## Farm CoreとFarm Instance

- **Farm Core**（このリポジトリ）：構造、スクリプト、ルール、テンプレート
- **Farm Instance**：実プロジェクト。このリポジトリをコピーし、`0_soil/` を自分の正史で埋める

Instanceの運用で「Coreにあるべきだった」と気づいたことは `FARM_CORE_FEEDBACK.md` に蓄積し、Coreへ還流します。

## アンチパターン（v1の実運用で確認された破綻）

1. **台帳を2つ以上持つ** — 変更ログ・索引・依存表を手で維持すると、必ず全部が腐る。履歴はgit、索引はスクリプト生成、依存は成果物自身のfrontmatterに置く
2. **昇格ラダーを深くする** — seeds→seedbank→nursery→…の多段昇格は、2段目から先が維持されない。seeds→soilの1段に保つ
3. **存在しない組織をロールプレイする** — 使用者が1人なら、仮想エージェント組織や`owner: marketing-team`のようなメタデータは演技であり、トークンと注意力を浪費する
4. **失敗をルールで修理する** — 生成事故のたびに散文ルールを足すと、ルール総量がAIの遵守能力を超える。修理は機構（検査スクリプト、構造変更）で行う
5. **正規ルートを近道より重くする** — 正しい手順が面倒なら、人間もAIも必ずバイパスする。バイパスされた時点で、フレームワークの保証はすべて消える

## クイックスタート

```bash
git clone <this-repo> my-project && cd my-project
# 1. 0_soil/identity.md を自分のプロジェクトで書き換える
# 2. AIに話す：「READMEとagent.mdを読んで。今日は◯◯について考えたい」
# 3. セッション終わりにAIが蒸留を提案する。承認してコミット
bash barn/farm.sh status   # いつでも現状確認
```

10分で最初のループが回ります。詳細なルールは `barn/rules.md`、AIへの指示は `agent.md` にあります。
