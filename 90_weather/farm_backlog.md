# Farm Backlog（運用改善候補）

> Farm 自体の運用で気づいた改善候補を、提案として整形して蓄積する場所。
> 詳細ルール：`40_trellis/rules/farm_evolution_rules.md` §1, §6, §9

未整形の気づきは `farm_reviews/` に置き、検討して提案として整形できた段階でここへ移す。
ここで採用が決まったものは `40_trellis/rules/` または `00_soil/farm/` に反映し、本ファイルから消すか `decision_log.md` への参照を残す。

## エントリ書式

```
- id: FB-YYYY-NNN
  date: YYYY-MM-DD
  source: <farm_reviews/<file>.md or 会話、レビュー等>
  category: <rules / structure / templates / prompts / docs / process>
  observed: <実際に起きたこと・困ったこと>
  risk: <放置した場合のリスク>
  proposal: <改善案>
  priority: <P0 / P1 / P2>
  status: <open / in_progress / adopted / rejected>
```
