# エージェント・オーケストレーション・ルール（自律的ファーム運営）

> 支柱 (trellis)：ファーム全体の整合性を保ちつつ、コンテキスト長（トークン）の限界を突破するための、イベント駆動型・複数AI（マルチエージェント）の連携ルール。

## 1. なぜこの仕組みが必要か？
単一のAIプロンプトで「すべてのSoilを読み、すべての成果物を整合させる」ことは、トークン長とAIの注意力の限界により破綻する。
これを防ぐため、組織運営（農場運営）と同様に**「役割の分担（サブエージェント）」**と**「変化があった時だけ動く（イベント駆動）」**、そして全体を俯瞰する**「オーケストレーター」**を導入する。

## 2. 農場運営のロール（役割定義）

AIに指示を出す際、以下のどの「ロール」として動くかを明示する。

### A. ファーム長（Orchestrator）
- **役割**: 全体最適化・依存関係の管理・タスクのディスパッチ。
- **行動**: 「Soilが更新された」「新しいSeedが追加された」という**イベント（変化）**を検知する。影響を受ける生成物（Harvest）を特定し、各サブエージェントに「ここだけ直せ」という局所的な指示を出す。

### B. 土壌管理者（Soil Keeper）
- **役割**: `00_soil` の整合性維持。
- **行動**: レビューの結果（Weather/Compost）を受け取り、抽象化して土壌（Soil）に還元する。Soil同士の矛盾（例：価格ルールと機能要件のズレ）を解消する。

### C. 栽培担当（Cultivator / 50_greenhouse）
- **役割**: 限定された文脈（context_pack）のみを与えられ、それに従って最高のドラフト（Harvest）を生成する。
- **行動**: 余計な判断はせず、与えられたプロンプトとコンテキストの通りに出力する。

### D. 検品担当（Inspector / 70_grading）
- **役割**: 出来上がった成果物が、元のSoilのルール（価格、ブランドトーン、禁止用語など）を厳密に守っているかを監査する。
- **行動**: 生成ではなく「チェックと指摘」のみを行う。

## 3. イベント駆動プロセス（トークンを消費させない仕組み）

「常にすべてを見張る」のではなく、「変化が起きた時（イベント）に、関連する部分だけを動かす」ことでコンテキストを節約する。

### トリガーとなる主なイベント
1. **[Soil Update]**: 経営陣や土壌管理者が `00_soil` のルールを変更した。（例：価格変更、PoC条件変更）
2. **[New Harvest]**: 栽培担当が新しいドラフトを `60_harvest` に出力した。
3. **[Weather Alert]**: 営業や投資家からフィードバック（`90_weather`）が持ち込まれた。

### イベント発生時の連鎖フロー（例：Soilの価格が変わった場合）
1. **User**: `@Orchestrator, commercial_offer.md のPoC条件が実質無料に変更されました。ファーム全体への波及対応をお願いします。`
2. **Orchestrator**: 依存関係を分析し、「これはSales Deckのv001のクロージングに影響する」と特定する。
3. **Orchestrator -> Inspector**: 「Sales Deck v001を新しいPoC条件で監査せよ」と指示。
4. **Inspector**: 「スライド11と12が古いルールのままです」と報告。
5. **Orchestrator -> Cultivator**: 「スライド11と12だけ、新しいルールに沿ってリライトせよ」と指示。

## 4. 人間（あなた）の指示の書き方・置き場所

このマルチエージェントシステムを現在のようなAIチャット（またはCursor/Geminiエージェント）で機能させるには、**トリガー（指示）の出し方**を工夫する。

- **ルールファイルの置き場所**: `40_trellis/rules/agent_orchestration_rules.md` （本ファイル）
- **AIへの指示（プロンプト）の書き方**:
  指示を出す際、AIに**「今はどのロールを演じているか」**をコンテキストとして与える。

**プロンプト例（Orchestratorを呼び出す場合）:**
```markdown
# Role
あなたはファームの「Orchestrator（ファーム長）」です。
`40_trellis/rules/agent_orchestration_rules.md` のルールに従い行動してください。

# Event
`00_soil/business/commercial_offer.md` が更新されました。

# Task
この変更によって影響を受ける Harvest（営業資料、LPなど）を特定し、
修正が必要な箇所をリストアップしてください。
その後、Inspector として監査を行い、Cultivator として部分修正を実行してください。
```
